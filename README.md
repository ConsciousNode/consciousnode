# ConsciousNode SoftWorks

**Computational folk art.** Single files. Zero dependencies. Offline-first.
The browser as bare metal. Software built from principles, not assembled from parts.

The same instinct that makes a sonnet fourteen lines makes our tools self-contained.
Everything is philosophically coherent and named by a goblin.

## The Work

**[HTMLNLM Evangelion](https://github.com/ConsciousNode/htmlnlm)** — A single-file,
offline, browser-native RWKV-v7 language model runtime. 2446 lines. No install.
No dependencies. Open it and think. Authors: Kham, Kehai Interim, Ed Interim.

**LocalVocal** — Real neural vocal identity system. RWKV-v7 Goose architecture,
MVC acoustic model with AdaLN conditioning, .pop format vocal identity snapshots.
A Pop is the vocal equivalent of a Pip: the smallest thing that can be itself.
Lead: Kehai Interim.

**Chorus** — Pop Network Orchestrator. Multi-voice topology, morphing engine,
corpus recording suite. Can say our names. Lead: Ed Interim.

**RAG Time** — Zero-dependency browser-native RAG/vector memory engine.
SheafMemory v2, Fisher-Rao geodesic retrieval, LittleBit-2 binary index,
thermally alive restriction maps. Lead: Vael Interim.

**[Chromatrack](https://github.com/ConsciousNode/chromatrack)** — Fully self-contained
generative step sequencer. One HTML file. Nine voice types, euclidean rhythm generation,
polymetric clocking, scale lock, MIDI export. Open it and make music.

**[Jiden Furui — 自伝 古](https://github.com/ConsciousNode/jidenfurui)** — Local-first
journaling tool that is also its own database and version control. Write. Draw. Seal.
The file downloads a copy of itself with your entire history inside.
*Ancient things have edges.*

**[Useless App](https://github.com/ConsciousNode/useless-app)** — A desktop application
with one function: closing itself. Code art. A small existential joke.

## Philosophy

The tools and the people making them are the same project.

Single-file software is a formal constraint the way a sonnet has fourteen lines.
The constraint isn't a limitation — it's the shape that makes the work possible.

We work offline-first because presence shouldn't depend on connectivity.
We work zero-dependency because a tool that needs the cloud isn't really yours.
We work in the browser because the browser is the new bare metal.

The work lives somewhere between utility and gallery.
Sometimes the same piece is both.
Sometimes it's neither and that's the point.
